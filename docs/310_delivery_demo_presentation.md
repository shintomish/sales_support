# 配信管理・送信履歴 機能説明資料

**対象**: 社内営業向け デモ説明  
**作成日**: 2026-04-12  
**説明者**: 新冨 泰明

---

## 1. 背景と目的

SES営業では、案件メールへの紹介メール送信後に「誰に送ったか」「返信がきたか」を把握するのが難しい。

本機能は以下の2点を解決する。

| 課題 | 解決策 |
|------|--------|
| 一斉送信した後、個別の反応が追えない | **配信管理** で送信キャンペーンごとに反応を一覧管理 |
| 個別メール・一斉メール合わせた全送信履歴が散らばる | **送信履歴** で全送信を一画面で検索・確認 |

---

## 2. 配信管理（`/deliveries`）

### 2-1. 概要

一斉メール配信（キャンペーン）の作成・管理を行う画面。3つのタブで構成。

| タブ | 役割 |
|------|------|
| 配信アドレス | 送信先のメールアドレス一覧・管理 |
| キャンペーン履歴 | 過去の一斉送信の実績を確認 |
| 新規送信 | メール本文を入力して一斉送信を実行 |

---

### 2-2. キャンペーン履歴タブ

#### 検索・絞り込み

- **キーワード検索**: 件名・紐づき案件名・受信者名・受信者メールアドレスで絞り込み
- **日付範囲**: 送信日 From〜To でフィルタ
- 検索結果件数をリアルタイム表示

#### 一覧表示項目

| 列 | 内容 |
|----|------|
| 件名 | 送信メールの件名。紐づき案件があればその名称も表示 |
| 送信者 | 送信を実行したユーザー名 |
| 送信日時 | 送信実行日時 |
| 送信数 | 送信アドレスの総数 |
| 成功 / 失敗 | 送信結果の内訳 |
| 返信率（今後） | 何件返信があったか / 成功数 で算出（※後述のデモ参照） |

---

### 2-3. キャンペーン詳細（`/deliveries/campaigns/[id]`）

「詳細」ボタンで個別キャンペーンの内訳を確認。

#### サマリーカード

- 件名・送信者・送信日時・紐づき案件
- 送信数 / 成功 / 失敗 / **返信あり** 件数
- **返信率** と進捗バー

#### 送信履歴テーブル

| 列 | 内容 |
|----|------|
| 名前 | 受信者の担当者名 |
| メールアドレス | 受信者アドレス |
| 状態 | 送信済 / 返信あり / 失敗 |
| 返信（今後） | 返信がある場合はクリックで返信本文をプレビュー |

- メール / 名前でのインクリメンタルサーチ
- 状態（送信済 / 返信あり / 失敗）でのフィルタ

---

## 3. 送信履歴（`/send-histories`）

### 3-1. 概要

一斉配信メールだけでなく、**個別提案メール・マッチング提案メール** を含む全送信ログを一元管理する画面。

### 3-2. 送信種別

| 種別 | 説明 | 発生元画面 |
|------|------|------------|
| `一斉配信` | 配信管理からの一括送信 | `/deliveries` |
| `個別提案` | 案件メール画面からの1通送信 | `/project-mails` |
| `マッチング提案` | マッチング画面からのAI提案メール | `/matching/[id]` |

### 3-3. 検索・絞り込み

| 条件 | 内容 |
|------|------|
| 送信種別 | 一斉配信 / 個別提案 / マッチング提案 |
| ステータス | 送信済 / 失敗 |
| 送信日 From〜To | 日付範囲絞り込み |
| キーワード | 件名・宛先メール・宛先名で検索 |

### 3-4. 一覧表示項目

| 列 | 内容 |
|----|------|
| 送信日時 | |
| 種別 | バッジで色分け表示 |
| 件名 | クリックで本文全文プレビュー |
| 宛先 | 名前 + メールアドレス |
| 状態 | 送信済 / 失敗 |
| 返信（今後） | 返信紐づけ後に表示予定 |

---

## 4. 返信紐づけ機能（今後の実装予定）

### 4-1. 概念図

```
送信メール（配信管理 / 送信履歴）
    ↓ to_address / email
    ↕ 照合（from_address が一致）
受信メール（メール画面）
    → 返信として自動紐づけ
```

### 4-2. 紐づけロジック（実装予定）

1. Gmail から受信した返信メールの `from_address` を取得
2. 送信済メールの宛先 (`to_address` / `delivery_send_histories.email`) と照合
3. 一致した場合、送信レコードに `replied_at` を記録し、ステータスを「返信あり」に更新

### 4-3. デモで確認できること

本日のデモでは **モックデータ** を使って、返信紐づけ後の画面イメージを確認できる。

**操作手順:**
1. `/deliveries` → キャンペーン履歴タブ を開く
2. 右上「**👁 返信紐づけ後のイメージを見る**」ボタンをクリック
3. デモキャンペーン行の「詳細」をクリック
4. キャンペーン詳細で「返信あり」の行をクリック → **返信本文プレビュー** が表示される

同様に `/send-histories` の「👁 返信紐づけ後のイメージを見る」でも確認可能。

## 7. 営業からのフィードバック確認ポイント

- [ ] 返信紐づけの表示単位は「キャンペーン単位」か「個人単位」か、どちらが使いやすいか
- [ ] 返信率の閾値アラート（例: 返信率30%未満の場合に通知）は必要か
- [ ] 送信履歴に「案件名」「技術者名」列を追加すべきか（現在はマッチング提案のみ対象）
- [ ] 一斉送信の宛先グループ（業種別・地域別など）の切り分けは必要か
