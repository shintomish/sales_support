# 技術者メール 追加機能説明書

> 対象読者：当社営業担当者  
> 追加日：2026-04-13

---

## 1. 追加機能まとめ

| 機能 | 内容 |
|------|------|
| 希望単価・年齢の自動抽出 | メール本文から希望単価（下限・上限）と年齢を自動抽出 |
| マッチ案件の単価フィルター | 技術者の希望単価に合う案件のみ表示（`案件単価上限 >= 希望単価`） |
| 低単価・単価未記載の自動除外 | 希望単価なし・35万未満は「除外」ステータスに自動分類 |
| 提案メールの件名統一 | `【技術者ご紹介】{案件名}（max{単価}万円）` 形式 |
| 提案メールへの年齢・署名挿入 | 本文に `K.K（35歳）氏` 形式＋メール署名設定を自動挿入 |
| タブ切替で右ペインリセット | タブ切替時に右ペインの選択がクリアされる |
| 送信履歴を配信管理に統合 | 全送信記録を配信管理（delivery_campaigns）で一元管理 |

## 2. 希望単価・年齢の自動抽出

### 抽出タイミング

- **新着取込**（初回スコアリング）で自動実行
- **全件再スコア**ボタンで既存レコードも再抽出

### 対応フォーマット例

| 本文例 | 抽出結果 |
|--------|---------|
| `■単金■\n60~70万円` | min=60, max=70 |
| `■単　金：110万（応相談）` | min=110, max=110 |
| `■単価：〜80万` | min=80, max=80 |
| `■希望単価：70〜90万` | min=70, max=90 |
| `MN(46歳/女性)` | age=46 |
| `S.N(28歳/男性)` | age=28 |
| `年齢：35` | age=35 |

> 「単」と「金」の間に全角スペースがある形式（`■単　金`）や、ラベルの後に改行して値が来る形式（`■単金■\n60~70万`）にも対応しています。

## 3. マッチ案件の単価フィルター

技術者の希望単価が設定されている場合、**案件単価上限 >= 技術者希望単価** の案件のみ表示します。

### 例（技術者希望単価 110万の場合）

| 案件 | 案件単価上限 | 表示 |
|------|------------|------|
| NWセキュリティプリセールスエンジニア | 123万 | 表示 ✓ |
| セキュリティ製品導入・設計支援 | 93万 | 非表示 |
| Javaバックエンドエンジニア | 75万 | 非表示 |

> 希望単価が未設定の場合は全案件を表示します。案件側に単価情報がない場合もその案件は表示されます。

---

## 4. 低単価・単価未記載の自動除外

新着取込のスコアリング時に以下の条件で自動除外されます。

| 条件 | 除外理由 |
|------|---------|
| 希望単価の記載がない | `no_unit_price` |
| 希望単価が 35万円/月 未満 | `unit_price_too_low` |

業務方針：「案件側の提示金額より安い単金の技術者を紹介する」ため、単価が極端に低い・不明な場合は紹介対象外としています。

## 5. 提案メールの仕様

### 件名

```
【技術者ご紹介】{案件タイトル}（max{案件単価上限}万円）
```

### 本文

- 冒頭1行目：`{名前}（{年齢}歳）氏のご紹介です。`（年齢未取得時は省略）
- 末尾：メール署名設定のテンプレートを自動挿入

---

## 6. 全件再スコア

全件再スコアボタンは、希望単価・年齢の抽出ロジック改善後に**過去データを一括更新**するために使います。

| | 変更前 | 変更後（2026-04-13） |
|---|---|---|
| 処理件数上限 | 500件 | **全件（上限なし）** |
| タイムアウト | 300秒 | 600秒 |

> 全件再スコアは添付ファイル解析をスキップするため、3,000件でも数十秒で完了します。

---

## 7. 送信履歴の統合

従来の「送信履歴」画面・`/send-histories` APIを廃止し、すべての送信記録を配信管理で管理します。

| send_type | 送信元 |
|-----------|--------|
| `delivery` | 案件メール → 一括配信 |
| `proposal` | 案件メール → 特定技術者への提案 |
| `matching_proposal` | マッチング画面からの提案 |
| `engineer_proposal` | 技術者メール画面からの提案 |
