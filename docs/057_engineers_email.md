# 技術者メール 追加機能説明書

> 対象読者：当社営業担当者  
> 作成日：2026-04-13

---

## 1. 追加機能まとめ

| 機能 | 内容 |
|------|------|
| 希望単価・年齢の自動抽出 | メール本文から希望単価（下限・上限）と年齢を自動抽出 |
| マッチ案件の単価フィルター | 技術者の希望単価以上の案件のみ表示 |
| 低単価・単価未記載の自動除外 | 希望単価なし・35万未満は「除外」に自動分類 |
| 提案メールの件名統一 | `【技術者ご紹介】{案件名}（max{単価}万円）` 形式 |
| 提案メール本文の改善 | 年齢付き氏名・メール署名を自動挿入 |
| 提案メール送信フォームの改善 | 担当者名（宛名）入力・ファイル添付（DnD対応） |
| タブ切替で右ペインリセット | タブ切替時に右ペインの選択をクリア |
| 送信履歴を配信管理に統合 | 全種別の送信記録を配信管理で一元管理 |
| 全件再スコアの上限撤廃 | 500件上限を廃止、全件対象に変更 |

## 2. 希望単価・年齢の自動抽出

### 抽出タイミング

- **新着取込**（初回スコアリング）で自動実行
- **全件再スコア**ボタンで既存レコードも再抽出

### 対応フォーマット

| 本文例 | 抽出結果 |
|--------|---------|
| `■単金■\n60~70万円` | min=60, max=70 |
| `■単　金：110万（応相談）` | min=110, max=110 |
| `■単金：60万（応相談）` | min=60, max=60 |
| `■単価：〜80万` | min=80, max=80 |
| `■希望単価：70〜90万` | min=70, max=90 |
| `MN(46歳/女性)` | age=46 |
| `S.N(28歳/男性)` | age=28 |
| `年齢：35` | age=35 |

**対応済みの特殊パターン：**
- `■単　金` — 「単」と「金」の間の全角スペース
- `■単金■\n60~70万` — ■終端＋改行後に値、半角チルダ `~` の範囲指定
- `MN(46歳/女性)` — 括弧内の性別より前にある年齢

### フォームからの手動入力

自動抽出できなかった場合は、右ペインの編集フォーム（希望単価 下限・上限）に手動で入力して保存できます。

## 3. マッチ案件の単価フィルター

技術者の希望単価が設定されている場合、**案件単価上限 >= 技術者希望単価** の案件のみ表示します。

```
表示条件：案件.unit_price_max >= 技術者の希望単価（unit_price_max または unit_price_min）
```

### 表示例（技術者希望単価 110万の場合）

| 案件 | 案件単価上限 | 表示 |
|------|------------|------|
| NWセキュリティプリセールスエンジニア | 123万 | 表示 ✓ |
| セキュリティ製品導入・設計支援 | 93万 | 非表示 |
| Javaバックエンドエンジニア | 75万 | 非表示 |

> 希望単価が未設定の場合は全案件を表示します。案件側に単価情報がない場合もその案件は表示されます。

## 4. 低単価・単価未記載の自動除外

新着取込のスコアリング時に以下の条件で自動的に「除外」ステータスになります。

| 条件 | 除外理由タグ |
|------|------------|
| 希望単価の記載がない | `no_unit_price` |
| 希望単価が 35万円/月 未満 | `unit_price_too_low` |

業務方針：「案件側の提示金額より安い単金の技術者を紹介する」ため、単価が極端に低い・不明な場合は紹介対象外としています。

> 全件再スコア時はこの除外ロジックは再適用されません（スコア点数のみでステータスを再判定）。

## 5. 提案メール送信フォーム

### 件名フォーマット

```
【技術者ご紹介】{案件タイトル}（max{案件単価上限}万円）
```

案件タイトル末尾に既に単価表示がある場合は除去して重複を防ぎます。

### 本文フォーマット

- 冒頭1行目：`{担当者名} 様`
- 2行目以降：AI生成の提案文（`{名前}（{年齢}歳）氏のご紹介です。` で始まる）
- 末尾：メール署名設定のテンプレートを自動挿入

### 送信フォームの項目

| 項目 | 内容 |
|------|------|
| 担当者名（宛名） | 入力すると本文冒頭の `○○ 様` をリアルタイムで書き換え |
| 宛先メール | 案件担当者のメールアドレス（自動入力・編集可） |
| 件名 | 上記フォーマットで自動生成・編集可 |
| 本文 | AIが生成・編集可 |
| 添付ファイル | ドラッグ＆ドロップまたはクリックで選択（複数可・最大10MB/件） |

## 6. 全件再スコア

抽出ロジック改善後に過去データを一括更新するためのボタンです。

| | 変更前 | 変更後（2026-04-13） |
|---|---|---|
| 処理件数上限 | 500件 | **全件（上限なし）** |
| タイムアウト上限 | 300秒 | 600秒 |
| 添付ファイル解析 | なし | なし（本文のみ） |

> 添付ファイル解析をスキップするため、3,000件以上でも数十秒で完了します。

---

## 7. 送信履歴の統合

従来の「送信履歴」画面・`/send-histories` APIを廃止し、すべての送信記録を**配信管理**（`delivery_campaigns` + `delivery_send_histories`）で一元管理します。

| send_type | 送信元 | 紐付け情報 |
|-----------|--------|-----------|
| `delivery` | 案件メール → 一括配信 | — |
| `proposal` | 案件メール → 特定技術者への提案 | engineer_id |
| `matching_proposal` | マッチング画面からの提案 | engineer_id / public_project_id |
| `engineer_proposal` | 技術者メール画面からの提案 | engineer_mail_source_id / public_project_id |

## 8. 追加されたDBカラム

| テーブル | カラム | 型 | 内容 |
|---------|--------|-----|------|
| `engineer_mail_sources` | `age` | smallint | 年齢 |
| `engineer_mail_sources` | `unit_price_min` | smallint | 希望単価（下限・万円/月） |
| `engineer_mail_sources` | `unit_price_max` | smallint | 希望単価（上限・万円/月） |
| `delivery_campaigns` | `send_type` | varchar(30) | 送信種別（delivery/proposal/matching_proposal/engineer_proposal） |
| `delivery_campaigns` | `engineer_mail_source_id` | bigint | 技術者メールとの紐付け |
| `delivery_send_histories` | `engineer_id` | bigint | 技術者マスタとの紐付け |
| `delivery_send_histories` | `public_project_id` | bigint | 案件との紐付け |
