## 1. 自動処理のスケジュール

システムは以下のタイミングで自動的にメールを処理します。

| 処理内容 | 実行タイミング |
|----------|--------------|
| 新着メールの取り込み（Gmail / KAGOYA IMAP それぞれ） | 15分ごと |
| 案件メール／技術者メールへの自動仕分け | 15分ごと |
| 案件メール画面への取込・スコアリング | 15分ごと |
| 技術者メール画面への取込・スコアリング | 15分ごと（100件/回） |
| 処理済みメールをGmailのゴミ箱へ移動 | 毎日 午前2時 |
| 古いメールの本文削除・レコード削除 | 毎日 午前3時 |

> システムが自動で動くため、担当者が手動で操作する必要はありません。
> ※ 旧バージョンにあった「AIによる抽出（30分毎）」は廃止し、案件・技術者それぞれの専用スコアリングに移行しています。

## 2. メールの保持期間とデータ削除ルール

サーバーの容量を圧迫しないよう、以下のルールで自動削除しています。

| 対象 | 経過期間 | 行われる処理 |
|------|----------|-------------|
| 「案件メール」または「技術者メール」に仕分けされたメール | 30日後 | メールの本文を削除（件名・差出人は残る） |
| 「案件メール」または「技術者メール」に仕分けされたメール | 90日後 | メール自体を完全削除 |
| 仕分けされなかったメール（未分類） | 14日後 | メール自体を完全削除（取りこぼしとみなす） |

### 削除後も案件履歴は残ります

メールが削除されても、**「誰から・どんな案件が届いたか」のメタ情報は「案件メール」画面に永続保存**されます。
メール本文が消えた後も、案件の受信履歴は引き続き確認できます。
