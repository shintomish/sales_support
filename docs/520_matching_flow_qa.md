## 1. 案件メールの「要確認」とは

### 判定の仕組み

受信した案件メールは、本文・件名のキーワードをもとに自動でスコアリングされます。
スコアに応じて以下の3つに振り分けられます。

| スコア | ステータス | 意味 |
|---|---|---|
| 60点以上 | **新着案件** | 案件確度が高いと判断。そのまま提案フローへ進める |
| 40〜59点 | **要確認** | 案件らしいが確信が持てない。人が判断する |
| 39点以下 | **除外** | 案件ではないと判断 |

### スコアの加点・減点ルール

**加点要素（最大85点）**

| 要素 | 加点 |
|---|---|
| 「案件ご紹介」「技術者募集」等の明示ワード | +15 |
| 「単価」「稼働開始」等の条件提示ワード | +10 |
| 技術スタック（言語・インフラ・DB） | 最大 +20 |
| 「XX万」という具体的な単価 | +15 |
| 勤務地（東京・大阪等） | +10 |
| 工程（要件定義 / 開発 / テスト等） | 最大 +10 |
| 即日・長期・人月 | +5 |

**減点（ペナルティ）**

| 要素 | 減点 |
|---|---|
| 「スキル見合い」「応相談」等の曖昧単価 | -10 |
| 「4次・5次・6次請け」等の高次商流 | -10 |

### 要確認になりやすいパターン

- 技術スタックや勤務地は記載されているが、明示的な案件紹介ワードや単価がない
- 案件の雰囲気はあるが条件が曖昧
- ペナルティが加わってスコアが下がった

## 2. 「要確認」と「新着案件」の違い

**違いはスコアのみ。** 中身の性質は同じ案件です。

| | 要確認 | 新着案件 |
|---|---|---|
| スコア | 40〜59点 | 60点以上 |
| 次のアクション | 「案件確定」か「除外」を人が判断 | そのまま提案フローへ |

### 案件処理フロー

```
メール受信
  ↓
自動スコアリング
  ├─ 60点以上 → 新着案件 → 提案済 → 面談 → 成約 / 失注
  ├─ 40〜59点 → 要確認  → 人が判断 → 「案件確定」で新着案件へ昇格 or 除外
  └─ 39点以下 → 除外
```

「要確認」は**人間のフィルタリングを挟むための中間ステータス**です。
「案件確定」ボタンを押すと「新着案件」に昇格し、通常の提案フローに乗ります。

## 3. 技術者管理への登録方法

### 現在の登録手順（手動）

技術者管理への登録は、サイドバーの「技術者管理」→「新規登録」から行います。
登録フォームは3タブ構成です。

#### 👤 基本情報（氏名のみ必須）

| 項目 | 必須 |
|---|---|
| 氏名 | ◎ 必須 |
| 氏名カナ | 任意 |
| メールアドレス | 任意 |
| 電話番号 | 任意 |
| 所属会社 | 任意（BP会社名など） |
| 所属担当者 | 任意（BP先の担当者名） |

#### 🔧 スキル（任意）

スキル名で検索して追加。各スキルに以下を設定します。

- **経験年数**（0〜50年、0.5刻み）
- **習熟度**（1:入門 / 2:基礎 / 3:実務 / 4:上級 / 5:エキスパート）

#### 📋 希望条件（任意）

| 項目 | 備考 |
|---|---|
| 希望単価 下限/上限 | 万円/月 |
| 稼働可能日 | 日付 |
| 希望勤務形態 | フルリモート / ハイブリッド / 出社 |
| 希望勤務地 | 東京都など |
| 自己PR | 得意分野・実績 |
| GitHub / ポートフォリオ URL | 任意 |
| マッチングマーケットに公開 | チェックで案件マーケットのマッチング対象になる |

> **氏名だけ入力すれば最低限登録できます。**

## 4. 案件メールから技術者管理への登録（現状と課題）

### 現状

案件メール画面から技術者管理へ直接登録する導線は**現時点では未実装**です。

現在の運用フロー：

```
技術者メール受信（添付スキルシートあり等）
  ↓
メール画面で本文・添付を確認（手動）
  ↓
技術者管理 → 新規登録 で手入力
```

### 今後の改善案

メール画面（技術者カテゴリのメール詳細）に「技術者として登録」ボタンを追加し、
差出人名・会社名・スキル情報を登録フォームに自動プリフィルする機能の実装を検討中。
