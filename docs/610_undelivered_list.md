# 不達メール一覧

配信メールの不達(バウンス)を検知し、`delivery_addresses` テーブルの `is_active = false` へ自動更新したレコードの週次ログ。

> **運用方針**: 週次で新しいセクション(## YYYY-MM-DD〜MM-DD 検出分)を先頭に追記する。古い記録は削除せず下に蓄積する。
> **関連**: 配信インフラ全体は [450_distribution_management.md](./450_distribution_management.md) 参照。

---

## 記録フォーマット(テンプレート)

新しい週次レポートを追加する際は、以下の雛形をコピーして最上位に挿入する。

```markdown
### 配信先一覧に不一致(X件) — 当システム外からの送信(無視可)

| # | 受信日時 (UTC) | 相手先メールアドレス | 実際の送信元 | 不達理由 |
|---|---|---|---|---|

### 不達理由の分類サマリ

| 理由 | 件数 |
|---|---|
| アドレス不存在 | - |
| 受信拒否 | - |
| メールボックス容量超過 | - |

| **合計** | - |

### 当週の考察・特記事項

- (必要に応じて記載)

---
```

---

## 不達理由の分類定義

| 理由 | 判別方法 | 対応 |
|---|---|---|
| **アドレス不存在** | Gmail の配信不能通知で「存在しないアドレス」「User unknown」等 | is_active=false 自動更新 |
| **受信拒否** | 「拒否されました」「blocked」「policy reject」等 | is_active=false 自動更新 |
| **メールボックス容量超過** | 「Quota exceeded」「mailbox full」等 | is_active=false 自動更新 |
| **SPF/DKIM 認証失敗** | Postmaster からの認証失敗通知 | 当システム送信分のみ対応。他アカウントからのバウンスは無視 |
