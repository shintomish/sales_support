# 案件メール フロー説明書

> 対象読者：当社営業担当者

---

## 1. 全体の流れ

```
Gmail受信
  ↓（自動・15分毎）
メール取込（emailsテーブルに保存）
  ↓（自動・15分毎）
技術者メール or 案件メール に自動分類
  ↓（自動・15分毎）
案件メール画面に表示・スコアリング
```

---

## 2. 「案件メール」に分類されるタイミング

**自動で15分毎に動きます。**

Gmailに新着メールが届いてから最大30分以内（取込15分 + 分類15分）に案件メール画面へ反映されます。

### 案件メールと判定される条件

技術者メールの条件（添付ファイルあり・【技術者情報】・人材系キーワード）に **いずれも該当しない** メールが案件メールになります。

### スコアリングも同時に実行

案件メールへの分類と同じ15分毎のバッチで、**スコアリングと情報抽出も自動実行**されます。  
技術者メールと異なり、スキルシートのダウンロードや Claude API 呼び出しは行わず、**正規表現のみ**で処理するため軽量で即時反映できます。

---

## 3. 各ボタンの動作

### 「新着取込」ボタン

画面右上の「新着取込」ボタンを押すと、未処理の案件メールを即時スコアリングします（自動実行と同じ処理）。

### 対象になるメール
- `category = project`（案件メールと分類済み）
- かつ、まだ案件メール画面に登録されていないもの（初回処理のみ）

### 対象から外れるメール
- すでに案件メール画面に一度でも表示されたことがあるメール

---

### 「全件再スコア」ボタン

**手動のみ。**

### 対象になるデータ
- 案件メール画面に登録されている **すべてのレコード**（新着・要確認・提案中・面談中・受注・失注・除外 すべて含む）

### 何が更新されるか
- 会社名・担当者名・スキル・単価・勤務地などの **抽出情報を再計算**
- スコアと **ステータス（新着 / 要確認 / 除外）を再判定**

> 💡 **いつ使うか：** 抽出ロジックを改善した後に、過去データも正しい情報に更新したいときに使います。

---

### 「全件再抽出」ボタン

**手動のみ。スコアは変更しません。**

### 何が更新されるか
- 抽出情報（会社名・担当者名・スキル・単価・勤務地など）のみ再計算
- スコアとステータスは **変わりません**

> 💡 **いつ使うか：** 抽出ロジックだけを直した後、スコアは保持したまま抽出情報を更新したいときに使います。

## 5. ステータスの意味と変更場所

### ステータス一覧

| ステータス | 意味 | スコア目安 |
|-----------|------|-----------|
| 新着 | 案件メールとほぼ確定（スコア高） | 60点以上 |
| 要確認 | 案件メールの可能性あり（要目視確認） | 40〜59点 |
| 除外 | 案件メールではないと判定 | 39点以下 |
| 提案中 | 技術者を提案した案件 | — |
| 面談中 | 面談が設定された案件 | — |
| 受注 | 成約した案件 | — |
| 失注 | 失注した案件 | — |

### ステータス変更できる場所

| ステータス | 変更できる場所 |
|-----------|--------------|
| 除外 | 右ペイン（レコード選択時）のステータスボタン |
| 提案中 | 右ペイン（新着 or 要確認の状態で選択時） |
| 面談中 | 右ペイン（提案中の状態で選択時） |
| 受注・失注 | 右ペイン（面談中の状態で選択時） |

## 6. スコア計算の詳細

スコアは **0〜100点** で自動計算され、点数によってステータスが決まります。

### ステータス判定基準

| スコア | ステータス | 意味 |
|--------|-----------|------|
| 60点以上 | **新着** | 案件メールとほぼ確定 |
| 40〜59点 | **要確認** | 案件メールの可能性あり・目視確認が必要 |
| 39点以下 | **除外** | 案件メールではないと判定 |

---

### スコア計算式

#### ステップ1：除外チェック（スコア計算前に強制除外）

以下に該当する場合はスコア0・除外確定となり、スコア計算は行いません。

| 除外条件 | 内容 |
|---------|------|
| 件名に除外ワード | 配信停止・メルマガ・広告・請求書・お支払い・正社員募集・中途採用・ご挨拶・お知らせ |
| 送信元アドレス | no-reply・noreply を含む |
| 自社ドメイン | aizen-sol.co.jp からのメール |

#### ステップ2：加点計算（最大85点設計）

| 区分 | 加点 | 条件 |
|------|------|------|
| **[A] 明示的案件ワード** | +15点 | 件名または本文に以下のいずれか1つ：案件ご紹介・要員ご紹介・技術者ご紹介・案件情報・要員紹介・技術者紹介・技術者募集・エンジニア募集・SE募集・PG募集 |
| **[B] 条件提示ワード** | +10点 | 件名または本文に以下のいずれか1つ：稼働期間・稼働開始・開始時期・参画時期・稼働率・単価・月額・工数 |
| **[C] 技術スタック** | 最大20点 | 言語1件目+10点、2件目+5点、インフラ+5点、DB+3点（合計20点上限） |
| **[D] 単価の具体性** | +15点 | 「○○万」という数字が含まれる |
| **[E] 勤務地** | +10点 | 東京・大阪・名古屋・横浜・品川・渋谷・新宿・福岡・仙台・札幌・在宅 のいずれかが含まれる |
| **[F] 工程** | 最大10点 | 上流工程（要件定義・基本設計・詳細設計）+10点、開発系（開発・実装・製造）+7点、その他（テスト・保守・運用）+4点 |
| **[G] 稼働・期間ワード** | +5点 | 即日・長期・人月 のいずれかが含まれる |

**ペナルティ（減点）**

| 区分 | 減点 | 条件 |
|------|------|------|
| 単価曖昧 | -10点 | 「スキル見合い」「応相談」が含まれる |
| 高次商流 | -10点 | 「4次」「5次」「6次」「7次」「8次」が含まれる |

**合計上限：85点**（実装上は0〜100にクランプ）

#### [C] 技術スタック対象キーワード

| カテゴリ | キーワード |
|---------|-----------|
| 言語 | Java / Spring / SpringBoot / PHP / Laravel / Python / Django / Flask / C# / .NET / JavaScript / TypeScript / React / Vue / Angular / Ruby / Rails / Go / Golang / Swift / Kotlin |
| インフラ | AWS / EC2 / RDS / S3 / Lambda / Azure / GCP / Docker / Kubernetes / Linux |
| DB | MySQL / PostgreSQL / Oracle / SQLServer / MongoDB / Redis |

#### ステップ3：ドメイン学習補正（±20点）

同じ送信元ドメインからの過去メールに判定データが5件以上ある場合、**案件率に応じて自動補正**されます。

| 案件率 | 補正 |
|--------|------|
| 80%以上 | +20点（信頼できる案件会社） |
| 20%以下 | -20点（案件メールをほぼ送ってこない） |
| その他 | 補正なし |

> フォームサービス（smoothcontact.com）・フリーメール（gmail.com 等）はドメイン補正の対象外です。

---

### スコア計算の具体例

**例：PHP・AWSの案件 + 60〜80万 + 東京 + 詳細設計**

| 区分 | 加点理由 | 点数 |
|------|---------|------|
| [A] | 「案件ご紹介」が件名に記載 | +15 |
| [B] | 「稼働開始」が記載あり | +10 |
| [C] | PHP(+10)・AWS(+5) | +15 |
| [D] | 「60〜80万」が記載あり | +15 |
| [E] | 「東京」が記載あり | +10 |
| [F] | 「詳細設計」が記載あり | +10 |
| [G] | 「長期」が記載あり | +5 |
| **合計** | | **80点 → 新着** |

### 判定理由タグの読み方

案件メール画面の各レコードに表示される **「判定理由」タグ** の意味：

| タグ表示例 | 意味 |
|-----------|------|
| `project_a:案件ご紹介` | [A] 案件ご紹介というワードで加点 |
| `project_b:稼働開始` | [B] 稼働開始というワードで加点 |
| `lang:PHP` | [C] PHPというスキルで加点（1件目） |
| `lang2:Python` | [C] Pythonというスキルで加点（2件目） |
| `infra:AWS` | [C] AWSで加点 |
| `db:MySQL` | [C] MySQLで加点 |
| `price_concrete` | [D] 単価の数字あり |
| `location:東京` | [E] 東京で加点 |
| `process:詳細設計` | [F] 詳細設計で加点 |
| `timing:即日` | [G] 即日で加点 |
| `penalty_vague:応相談` | ペナルティ：単価曖昧 |
| `penalty_chain:4次` | ペナルティ：高次商流 |
| `domain:example.co.jp:+20(95%/22件)` | ドメイン補正：案件率95%・22件実績で+20点 |
| `excluded` | 除外ワードに該当してスコア0 |

## 7. 自動抽出される情報一覧

案件メールから正規表現で以下の情報が自動抽出されます。右ペインで手動修正も可能です。

| 項目 | 内容 |
|------|------|
| 会社名 | 送信元の会社名（メール本文・署名・from名から抽出） |
| 担当者名 | 送信元の担当者名 |
| 電話番号 | 担当者の電話番号 |
| 案件タイトル | 件名から【】を除いたもの |
| 必須スキル | 技術スタックキーワードの一致リスト |
| 工程 | 要件定義・開発・テストなど |
| 勤務地 | 勤務地・最寄駅（都市名から抽出） |
| リモート可否 | フルリモート可 / 不可 / 不明 |
| 単価（下限） | 「○○万〜」の下限値（万円） |
| 単価（上限） | 「〜○○万」の上限値（万円） |
| 稼働開始 | 「即日」「2026-5」など |
| 契約形態 | 準委任・派遣・請負 |
| 年齢上限 | 「〜40歳」など |
| 外国籍可否 | 可 / 不可 / 不明 |
| 商流 | 元請（1次）・2次・3次 |
